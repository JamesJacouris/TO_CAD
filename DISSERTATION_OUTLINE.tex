\begin{abstract}
% ~150 words | structured: problem → approach → results → contribution
% ---------------------------------------------------------------------------
Topology optimisation methods such as SIMP produce structurally efficient
material distributions, but their voxelised density-field outputs cannot be
directly edited, dimensioned, or manufactured.
This dissertation presents a fully automated, open-source Python pipeline that
converts 3-D SIMP topology optimisation results into editable parametric CAD
models.
The pipeline introduces: (i) a multi-signal zone classifier combining PCA shape
analysis, aspect ratio, boundary condition tag density, and EDT uniformity to
distinguish beam-like from plate-like structural regions automatically;
(ii) iterative size-then-layout frame optimisation, achieving 50--60\%
compliance reduction and demonstrated 2-iteration convergence---a finding not
previously reported in the literature; and
(iii) end-to-end automation from the Top3D SIMP solver through skeleton
extraction, graph optimisation, and FreeCAD parametric model export.
All four benchmark test structures are correctly classified.
The system is validated on a $150 \times 40 \times 4$ cantilever case matching
Yin's \cite{yin2020} original benchmark and is released as open-source software.
\end{abstract}

\section{Introduction}
\label{sec:intro}
% ~1 page
% Goal: explain the problem clearly to any reader, state contributions crisply.
% ---------------------------------------------------------------------------

Topology optimisation (TO) is a computational method for finding the most
efficient distribution of material within a design domain for a given set of
loads and constraints.
Given the geometry of the space, the boundary conditions, and a volume budget,
the algorithm asks: \emph{where should material be placed?}
The answer it produces is not a drawing or a model --- it is a \emph{density
field}: a number between 0 and 1 assigned to every element in the mesh,
indicating how much material should occupy that region.
Figure~\ref{fig:intro_gap} illustrates this output for a classic cantilever
problem: a three-dimensional grid in which elements range from solid (white)
to void (transparent), tracing efficient load paths between the support and
the applied load.

\noindent
A density field cannot be directly manufactured or edited.
An engineer who wants to build the optimal structure shown on the left of
Figure~\ref{fig:intro_gap} must first translate it into explicit geometry:
deciding which regions are beams, which are plates, where members connect,
and what their cross-sections should be.
For a three-dimensional domain this is a slow, subjective, and error-prone
process that can take days and that often discards the structural efficiency
identified during optimisation.
This \emph{interpretation gap} is the central problem this dissertation
addresses.

\subsection{Approach}
\label{subsec:approach}

% Explain, in plain English, what the pipeline does — before any terminology.
% A reader who has never heard of skeletonisation should understand this.

This work develops a fully automated Python pipeline that takes a SIMP density
field and produces a parametric CAD model ready for editing in FreeCAD.
The pipeline works in four steps, shown schematically in
Figure~\ref{fig:pipeline_overview}:

\begin{enumerate}
  \item \textbf{Topology optimisation.}
        The SIMP solver (Top3D \cite{liu2014}) runs on the user-specified
        domain, producing a density field and a record of which elements
        are near supports or loads.

  \item \textbf{Skeletonisation and graph extraction.}
        The density field is thresholded to a binary solid, then iteratively
        ``thinned'' until only a one-voxel-wide skeleton remains --- a line
        drawing that traces the centrelines of every structural member.
        This skeleton is converted into a graph: nodes at junctions and
        endpoints, edges along member centrelines.

  \item \textbf{Zone classification.}
        Connected regions are automatically classified as either
        \emph{beam-like} (slender, one-dimensional) or
        \emph{plate-like} (flat, two-dimensional) using four geometric
        signals combined by a weighted score.
        Beam regions are thinned to curves; plate regions are thinned to
        surfaces and extracted separately.

  \item \textbf{Frame optimisation and CAD export.}
        Beam cross-section radii are optimised by the Optimality Criteria
        method; node positions are then refined by L-BFGS-B.
        These two steps alternate until convergence.
        The final graph is exported as a parametric FreeCAD macro, with
        circular beam members swept along cubic Bézier centrelines.
\end{enumerate}

\subsection{Contributions}
\label{subsec:contributions}

This dissertation makes four specific contributions beyond the prior art:

\begin{enumerate}
    \item \textbf{Open-source Python pipeline.}
          A complete reimplementation of Yin's \cite{yin2020} MATLAB
          topology-optimisation-to-CAD pipeline in Python, with boundary
          condition (BC) tags propagated through every stage so that support
          and load points are never destroyed by skeletonisation or graph
          simplification.

    \item \textbf{Multi-signal zone classification.}
          A classifier that combines four geometric signals --- PCA shape,
          aspect ratio, BC tag density, and EDT thickness uniformity --- to
          automatically distinguish beam-like from plate-like regions in 3-D
          density fields before thinning.
          This is the first such classifier reported for this reconstruction
          problem class.

    \item \textbf{Iterative optimisation convergence analysis.}
          An empirical demonstration that two iterations of alternating
          size-then-layout frame optimisation capture $>95\%$ of the total
          obtainable compliance reduction, with the ordering (size before
          layout) being critical to monotonic convergence.
          No prior work reports this result.

    \item \textbf{End-to-end automation.}
          The complete pipeline runs from a SIMP density field to a
          FreeCAD macro via a single command-line call, including curved-beam
          fitting, plate extraction, and beam-to-plate joint creation.
\end{enumerate}

\subsection{Topology Optimisation and Its Output}
\label{subsec:simp_background}

% Keep this short — the reader needs to understand what TO produces, not a
% full survey of TO methods. One paragraph on SIMP is enough.
% Key points to make:
%   - SIMP assigns a density rho_e in [0,1] to every element
%   - Compliance C = F^T u is minimised subject to a volume constraint
%   - The density filter and OC update produce a converged density field
%   - The output is a NUMBER per element, not a shape — this is the root of
%     the interpretation problem

The SIMP method \cite{bendsoe1988,sigmund2001} minimises structural compliance
$C = \mathbf{F}^\top\mathbf{u}$ by varying element densities
$\rho_e \in [\rho_{\min}, 1]$ subject to a volume constraint.
Material stiffness is interpolated as
$E(\rho_e) = E_{\min} + \rho_e^p(E_0 - E_{\min})$,
where $p = 3$ penalises intermediate densities, driving the solution toward
0--1 distributions that resemble discrete material layouts.
A density filter prevents checkerboarding; an Optimality Criteria (OC)
update satisfies the volume constraint at each iteration.
The 3-D extension Top3D \cite{liu2014} implements this in 169 lines of MATLAB
using 8-node hexahedral elements.

The result is a density value per finite element --- not a shape.
Converting this into manufacturable, editable geometry is a non-trivial
post-processing problem, and it is the problem this dissertation addresses.

\subsection{Reconstruction Approaches}
\label{subsec:reconstruction_survey}

% Three clear sub-approaches: surface fitting, lofting along skeletons,
% primitive CSG from skeletons. Explain each in terms a non-expert can follow.
% Anchor the comparison in Table 1 (the comparison table).
% For each approach: what does it do, what does it produce, what does it miss.

\subsubsection{Surface fitting}

The simplest approach is to fit smooth surfaces directly to the outer boundary
of the density field.
NURBS or B-spline patches are fitted to the faceted mesh produced by
thresholding and marching cubes, yielding a smooth, closed solid.
Commercial tools (Altair Inspire, Ansys Discovery) follow this strategy, as do
academic methods by Park et al.\ \cite{park_surface_2019} and more recently
Beccari et al.\ \cite{beccari_topological_2025}.

The limitation is topological: NURBS patches describe smooth manifolds and
struggle with the branching, intersecting geometry that TO routinely produces.
Patch stitching at junctions requires manual intervention, and the resulting
model encodes no information about the internal load path --- only the outer
shape.
For simple, well-separated geometries surface fitting is adequate; for
general TO results it is not reliable.

\subsubsection{Skeleton-based reconstruction with lofting}

A more principled approach extracts the \emph{skeleton} of the density field
--- a one-voxel-wide or one-element-wide curve that traces the centreline of
every structural member --- and then reconstructs geometry \emph{around} that
skeleton rather than fitting to the outer boundary.

Amroune et al.\ \cite{amroune2021} extract a curve skeleton by contracting the
surface mesh toward its medial axis (Laplacian contraction \cite{au_skeleton_2008}),
then generate smooth branch surfaces by sweeping cross-sections along each
skeleton edge and blending them at junctions.
Feng et al.\ \cite{feng2025} improve junction continuity by computing
Rotation Minimising Frames (RMF) along each branch before sweeping, producing
STEP-format output compatible with commercial CAD.

Both methods produce high-quality surface geometry for individual branches.
Their limitation is that they perform no structural re-optimisation after
reconstruction, and neither can represent plate-like regions --- broad, flat
areas that a beam network cannot adequately describe.

\subsubsection{Primitive CSG from skeletons}

% This is the approach this work follows. Explain it carefully.
% Key ideas: homotopic thinning preserves topology; graph nodes = joints;
% edges = beams; CSG = cylinders + spheres. Then Yin adds size/layout opt.
% Polak adds refinement. This work adds hybrid classification + convergence.

Yin \cite{yin2020} established the most complete skeleton-based pipeline:
the density field is voxelised, iteratively thinned to a one-voxel-wide
skeleton using homotopic parallel directional thinning \cite{bertrand1994},
converted into a weighted graph, and then reconstructed as a
Constructive Solid Geometry (CSG) model of cylinders and spheres.
The thinning step is \emph{topology-preserving}: it removes only
\emph{simple points} --- voxels whose deletion leaves both the foreground
(solid) and background (void) connectivity unchanged --- so the skeleton
has the same number of loops and branches as the original solid.

Crucially, Yin's pipeline also includes structural re-optimisation.
After graph extraction, beam cross-section radii are optimised by the
Optimality Criteria method (size optimisation) and node positions by
gradient-based layout optimisation, recovering much of the structural
efficiency lost during skeletonisation.

% Now explain what Yin misses, motivating this work.
% Three gaps: beam-only, no convergence analysis, MATLAB only.

Yin's pipeline has three limitations.
First, it produces beam-only models: every structural region is thinned to a
curve and reconstructed as a cylinder.
Plate-like regions --- which arise naturally in TO results for distributed
loads, roof slabs, and panels --- are incorrectly reduced to beam networks.
Second, the size and layout optimisation steps are run once each, with no
analysis of whether further iterations would improve the result.
Third, the implementation is in MATLAB and is not publicly available.

Polak et al.\ \cite{polak2023} extended Yin's approach with an evolutionary
refinement loop and FreeCAD export, but without formal optimisation or
convergence guarantees.
The refinement is heuristic: branch lengths and node positions are adjusted
by rules rather than by minimising a structural objective.

\subsection{Pipeline Overview}
\label{subsec:overview}

The pipeline comprises four automated stages:
\textbf{Stage 0} (Top3D) produces the SIMP density field and BC tag arrays;
\textbf{Stage 1} (Reconstruction) thresholds, thins, and extracts a graph;
\textbf{Stage 2} (Size optimisation) assigns radii by OC;
\textbf{Stage 3} (Layout optimisation) repositions nodes by L-BFGS-B.
Stages 2 and 3 iterate until convergence.
BC tags (support = 1, load = 2) are generated in Stage 0 and propagated
unchanged through all downstream stages, ensuring structural meaning is
preserved at every point.

\subsection{Topology Optimisation (Stage 0)}
\label{subsec:top3d}

The compliance minimisation problem is
\begin{equation}
    \min_{\boldsymbol{\rho}} \; C = \mathbf{F}^{\top}\mathbf{u}, \quad
    \text{s.t.} \; \mathbf{K}(\boldsymbol{\rho})\mathbf{u} = \mathbf{F},\;
    \sum_e \rho_e v_e \leq V^*,\; \rho_{\min} \leq \rho_e \leq 1,
    \label{eq:simp_opt}
\end{equation}
where stiffness follows $E(\rho_e) = E_{\min} + \rho_e^p(E_0 - E_{\min})$
with $p = 3$.
Element sensitivities are
\begin{equation}
    \frac{\partial C}{\partial \rho_e}
      = -p(E_0 - E_{\min})\rho_e^{p-1}\,\mathbf{u}_e^{\top}\mathbf{K}_E\mathbf{u}_e.
    \label{eq:simp_sens}
\end{equation}
A distance-weighted density filter with radius $r_{\min}$ prevents
checkerboarding; the OC bisection update with move limit 0.2 satisfies the
volume constraint.
Convergence criterion: $\max_e |\rho_e^{\text{new}} - \rho_e^{\text{old}}| < 0.01$.

\textbf{BC tag generation (novel extension).}
Elements adjacent to fixed-DOF nodes are tagged \texttt{bc\_tag = 1} (support);
elements adjacent to loaded nodes are tagged \texttt{bc\_tag = 2} (load).
Without tags, thinning can destroy load-introduction points, rendering the
reconstructed frame structurally meaningless.

\subsection{Skeletonisation and Graph Extraction (Stage 1)}
\label{subsec:skel}

\subsubsection{Binarisation and Parallel Directional Thinning}

The density field is thresholded at \texttt{vol\_thresh} (default 0.3) to
produce a binary voxel grid.
Yin's parallel directional thinning \cite{yin2020} iteratively removes
\emph{simple points}---voxels whose deletion preserves both foreground
26-connectivity and background 6-connectivity---in six direction sweeps
($\pm X, \pm Y, \pm Z$).
End voxels (exactly one 26-neighbour) and BC-tagged voxels are protected from
deletion.
The algorithm is guaranteed to converge by monotonic voxel removal and
terminates when no candidate simple points remain, typically in 8--12 iterations
for domains under $100^3$ elements.
All inner loops are compiled with Numba \texttt{@njit} for performance.

\begin{algorithm}[htbp]
\caption{Parallel directional thinning with BC protection}
\label{alg:thinning}
\begin{algorithmic}[1]
\Repeat
  \For{each direction $d \in \{+X,-X,+Y,-Y,+Z,-Z\}$}
    \State $\mathcal{C} \gets$ border voxels facing direction $d$
    \State Remove from $\mathcal{C}$: end voxels, BC-tagged voxels,
           non-simple points
    \ForAll{$v \in \mathcal{C}$} \Comment{sequential with re-verification}
      \If{$v$ is still a simple point}
        \State delete $v$
      \EndIf
    \EndFor
  \EndFor
\Until{no deletions in full sweep}
\end{algorithmic}
\end{algorithm}

\subsubsection{Graph Construction and Post-Processing}

Skeleton voxels with $\neq 2$ foreground neighbours are designated \emph{feature
voxels}: those with 1 neighbour are end nodes; those with $>2$ are junction
nodes.
BC-tagged clusters are replaced by their centroid (connected component analysis)
and reconnected via 3-D Bresenham line drawing, preventing spurious graph
fragmentation.
Marching from each feature voxel through regular voxels yields graph edges with
intermediate polyline waypoints.

Post-processing applies four operations in sequence:
(1) edge collapse (merge nodes on edges shorter than \texttt{collapse\_thresh});
(2) branch pruning (remove dangling branches shorter than \texttt{prune\_len},
protecting BC-tagged terminations);
(3) RDP geometric simplification \cite{ramer1972} on waypoints;
(4) radius estimation via uniform volume-matching
$r = \sqrt{V_{\text{target}} / (\pi L_{\text{total}})}$ (preferred over
EDT-median because it preserves total volume).

\subsection{Frame Optimisation (Stages 2--3)}
\label{subsec:frame_opt}

\subsubsection{FEM Formulation}

Each beam is a 3-D Euler-Bernoulli element with 12 DOF (6 per node: 3
translational + 3 rotational).
For a circular cross-section of radius $r$:
$A = \pi r^2$, $I_y = I_z = \tfrac{\pi r^4}{4}$, $J = \tfrac{\pi r^4}{2}$.
The $12\times12$ local stiffness matrix incorporates axial, bi-planar bending,
and torsional stiffness, transformed to global coordinates via a rotation
matrix (with a special case for vertical members).
Global assembly uses sparse COO format; the system is solved by
\texttt{scipy.sparse.linalg.spsolve}.

\subsubsection{Size Optimisation (OC)}

Design variables: beam radii $\{r_i\}$.
Objective: minimise compliance $C = \mathbf{u}^{\top}\mathbf{K}\mathbf{u}$.
Volume constraint: $\sum_i \pi r_i^2 L_i = V_{\text{target}}$.
Analytical sensitivities:
\begin{equation}
    \frac{\partial C}{\partial r_i}
      = -\mathbf{u}_e^{\top}\frac{\partial \mathbf{K}_e}{\partial r_i}\mathbf{u}_e,
    \quad
    \frac{\partial A}{\partial r} = 2\pi r,\;
    \frac{\partial I}{\partial r} = \pi r^3,\;
    \frac{\partial J}{\partial r} = 2\pi r^3.
\end{equation}
The OC update with Lagrange multiplier bisection and move limits converges in
15--25 iterations.

\subsubsection{Layout Optimisation (L-BFGS-B)}

Design variables: $xyz$ coordinates of free (non-BC-tagged) nodes.
Bounds: node positions constrained to the domain envelope offset by maximum
radius.
Gradients computed by finite differencing (central differences, $\varepsilon =
10^{-4}$); \texttt{scipy.optimize.minimize} with L-BFGS-B.
Post-optimisation node snapping via Union-Find merges nodes within
\texttt{snap} distance.

\subsubsection{Iterative Convergence}
\label{subsubsec:convergence}

\textbf{Key empirical finding.}
Size optimisation followed by layout optimisation (Size $\to$ Layout) produces
monotonically decreasing compliance.
The reverse order (Layout $\to$ Size) causes compliance spikes and is
structurally counterproductive.
Across all test cases:
\begin{itemize}
    \item \textbf{Iteration 1:} Size achieves $\approx50\%$ compliance
          reduction; Layout achieves a further $\approx20\%$.
    \item \textbf{Iteration 2:} Additional 3--5\% improvement.
    \item \textbf{Iterations $\geq$3:} Diminishing returns $<1\%$.
\end{itemize}
Two iterations are therefore sufficient for engineering practice.
No prior work \cite{yin2020,polak2023} reports this convergence behaviour.

\subsection{Hybrid Beam-Plate Classification and Extraction}
\label{subsec:hybrid}

Topology optimisation results for distributed-load problems (roof slabs,
panels) contain both beam-like members and plate-like regions.
Representing plates as beam networks is structurally inaccurate and
geometrically poor.
The hybrid pipeline classifies connected voxel regions \emph{before thinning}
(classification requires thickness and shape information that thinning
destroys).

\subsubsection{Multi-Signal Zone Scoring}

Four signals are computed per region and combined with tuned weights:

\begin{enumerate}
    \item \textbf{PCA shape (0.3):}
          Eigenvalues $\lambda_1 \geq \lambda_2 \geq \lambda_3$ from PCA of
          voxel coordinates give
          $\text{Planarity} = 1 - \lambda_3/\lambda_2$ (high for plates) and
          $\text{Linearity} = 1 - \lambda_2/\lambda_1$ (high for beams).

    \item \textbf{Aspect ratio (0.3):}
          $\min(\lambda_1,\lambda_2)/\max(\lambda_1,\lambda_2)$---plates $\approx
          1.0$, beams $\ll 1.0$.

    \item \textbf{BC tag density (0.2):}
          Fraction of load-tagged voxels biases toward plate; fraction of
          support-tagged voxels biases toward beam.

    \item \textbf{EDT uniformity (0.2):}
          Coefficient of variation of EDT values within the region---plates are
          uniform (low CoV); beams taper (high CoV).
\end{enumerate}

$\text{plate\_score} = \text{weighted sum}$;
regions with plate\_score $> 0.5$ are plate zones, otherwise beam zones.
For large regions ($>2000$ voxels), block-level $12^3$ sub-analysis is used.

\subsubsection{Separate Thinning and Plate Extraction}

Beam zones use curve-preserving thinning (mode 0); plate zones use
surface-preserving thinning (mode 1).
Interface boundary voxels are masked to prevent skeleton retraction across
zone boundaries.
Each plate region produces a solid mesh (marching cubes) and a mid-surface with
per-vertex thickness $t = 2 \times \text{EDT} \times \text{pitch}$.
Beam endpoints are snapped to plate boundary vertices via KD-tree search to
create beam-to-plate connectivity.

\subsection{CAD Export}
\label{subsec:cad}

The optimised graph is exported as a Python macro for FreeCAD.
Straight beams use \texttt{Part.Cylinder}; curved beams use cubic Bézier
centreline paths fitted by least-squares to skeleton waypoints, swept via
\texttt{makePipeShell} with Frenet mode.
Control point sanitisation enforces chord monotonicity and limits perpendicular
bulge to prevent self-intersecting curves.
Every geometry call is wrapped in \texttt{try/except} with graceful
skip-and-log; \texttt{FreeCADGui.updateGui()} is called every 20 elements to
prevent GUI freeze.

\subsection{Iterative Convergence Analysis}
\label{subsec:convergence_results}

\figstub{Compliance convergence plot: $x$-axis = pipeline stage
  (Stage-1, Size-1, Layout-1, Size-2, Layout-2, \ldots),
  $y$-axis = compliance value. Three lines: opt\_loops = 1, 2, and 5.
  \TODO{Generate from pipeline history JSON.}}{
  Compliance reduction across iterative size-then-layout optimisation loops.
  Diminishing returns after iteration~2 are clearly visible.}

The convergence plot confirms that 2 iterations capture $>95\%$ of total
compliance reduction obtainable in 5 iterations.
The ordering effect (Size $\to$ Layout vs.\ Layout $\to$ Size) is
structurally significant: reversing the order causes compliance to increase
after layout optimisation in the first iteration, because the layout solver
moves nodes away from load paths before radii have been calibrated to the
current geometry.
This suggests that size optimisation should always precede layout optimisation
to provide a consistent stiffness landscape for node repositioning.

\subsection{Limitations}
\label{subsec:limitations}

\begin{itemize}
    \item \textbf{No shell FEA for plates.}
          Plate regions are classified and extracted but excluded from the
          frame FEA model.
          Structural validation of hybrid models requires shell element
          integration (future work).

    \item \textbf{Single load case.}
          The OC and L-BFGS-B formulations handle one load case; multi-case
          extension requires a weighted compliance sum over load scenarios.

    \item \textbf{No manufacturing constraints.}
          The pipeline produces geometrically valid CAD but does not enforce
          overhang angles, minimum feature sizes, or additive manufacturing
          build-direction constraints.

    \item \textbf{Computational cost scaling.}
          Top3D dominates runtime for large domains ($>100^3$ elements);
          thinning and zone classification have not been profiled beyond
          $100^3$ and may require GPU acceleration for production use.
\end{itemize}

\label{sec:conclusions}

\subsection{Summary}

This dissertation presented a fully automated, open-source Python pipeline
converting 3-D SIMP topology optimisation results into editable parametric
FreeCAD models.
Four contributions were made and validated:

\begin{enumerate}
    \item A \textbf{Python pipeline} matching Yin's \cite{yin2020} beam-only
          benchmark, extended with BC tag propagation and open-source release.
    \item A \textbf{multi-signal zone classifier} correctly identifying all
          four benchmark structures---including ambiguous cases that single-metric
          approaches fail on.
    \item \textbf{Iterative convergence analysis} showing that 2 iterations
          of Size $\to$ Layout optimisation capture $>95\%$ of obtainable
          compliance reduction, with ordering (size before layout) critical to
          monotonic convergence.
    \item \textbf{End-to-end automation} from raw density fields to
          curved-beam parametric CAD models, validated on benchmark test cases.
\end{enumerate}

This is a complete, working, open-source system, not a theoretical proposal.
The 50--60\% compliance reduction achieved by the frame optimisation
demonstrates that the reconstructed model is not merely geometric
approximation but a structurally refined design.

\subsection{Future Work}

\begin{itemize}
    \item \textbf{Shell finite elements for plates:} Integrate shell element
          formulation to enable true hybrid beam-shell structural optimisation
          within the existing pipeline.
    \item \textbf{Analytical layout gradients:} Replace finite-difference
          Jacobians in L-BFGS-B with analytical adjoint gradients
          ($\partial C / \partial \mathbf{x}_i = -\mathbf{u}^\top
          (\partial \mathbf{K}/\partial \mathbf{x}_i)\mathbf{u}$),
          reducing layout optimisation cost by $\mathcal{O}(3N)$ FEM solves
          per iteration.
    \item \textbf{Multi-load-case and manufacturing constraints:}
          Extend the optimisation formulation to weighted multi-case compliance
          and incorporate overhang/minimum-feature constraints for additive
          manufacturing.
    \item \textbf{Commercial CAD integration:}
          Extend export beyond FreeCAD macros to STEP/IGES via
          pythonOCC for compatibility with SolidWorks and Fusion 360.
\end{itemize}
